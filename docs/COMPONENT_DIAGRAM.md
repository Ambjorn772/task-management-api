## Опис компонентів

### API Router

- **Відповідальність**: Маршрутизація HTTP запитів
- **Технологія**: Express.js
- **Взаємодія**: Приймає запити, передає в Controller

### Controller

- **Відповідальність**: Обробка бізнес-логіки, координація між компонентами
- **Взаємодія**: Використовує Validation Service, Repository, Tasks Client (для Users Service)

### Validation Service

- **Відповідальність**: Валідація вхідних даних
- **Взаємодія**: Викликається з Controller перед обробкою даних

### Repository

- **Відповідальність**: Абстракція доступу до даних
- **Взаємодія**: Використовує Model для роботи з даними

### Model

- **Відповідальність**: Визначення структури даних
- **Взаємодія**: Використовується Repository для зберігання/отримання даних

### Tasks Integration Client (тільки в Users Service)

- **Відповідальність**: HTTP клієнт для комунікації з Tasks Service
- **Взаємодія**: Викликається з Controller для отримання завдань користувача
